## 项目结构分析

### 1. Eastmoney 分析师数据采集系统

- __功能__：自动采集东方财富网分析师排名数据，提取跟踪成份股信息
- __技术栈__：Python + Selenium + Chrome浏览器
- __数据处理__：采集HTML页面，提取结构化数据，生成Excel和HTML报告
- __运行方式__：通过cron定时任务，每天17:00执行

### 2. Stock Monitor 股票监控系统

- __功能__：实时监控股票价格，设置高低价格警报、涨跌幅警报、关键价位警报等
- __技术栈__：Python + Flask Web框架
- __前端__：提供Web管理界面，可配置监控股票和报警条件
- __数据源__：通过API获取股票数据
- __通知机制__：支持多种通知方式，可设置全局和单个股票的开关
- __运行方式__：通过cron每分钟检查一次，同时启动Web服务器(端口5001)

### 3. News 新闻爬虫系统

- __功能__：爬取热点新闻数据，生成分析报告
- __技术栈__：Python + Requests
- __通知机制__：支持飞书、钉钉、企业微信、Telegram、邮件、ntfy等多种通知渠道
- __运行方式__：通过cron每4小时执行一次

### 4. IPTV 频道列表下载系统

- __功能__：从配置文件中定义的URL下载M3U格式的电视频道列表
- __运行方式__：通过cron每天凌晨1点执行

## 容器配置和运行机制

### Dockerfile配置：

- 基础镜像：Ubuntu 22.04

- 安装了Chrome浏览器和ChromeDriver用于网页抓取

- 安装了Nginx作为Web服务器

- 配置了多个cron定时任务：

  - 每天凌晨1点运行IPTV脚本
  - 每4小时运行新闻爬虫
  - 每天17点运行东方财富分析师数据采集
  - 每分钟检查股票监控

### Nginx配置：

- 端口80映射到主机4400端口
- 配置了新闻目录(/data/news/)的访问路径
- 配置了IPTV M3U文件的访问路径

### 数据存储：

- 所有数据存储在/data目录下，通过卷挂载到主机的/vol1/1000/docker/my-ubuntu/run目录
- 包含新闻数据、股票监控数据、IPTV数据等

## 运行流程

容器启动后会：

1. 启动cron服务来执行定时任务
2. 启动nginx服务提供Web访问
3. 股票监控系统通过check_stock_monitor.sh脚本确保运行状态

这个项目是一个综合性的金融数据监控系统，集成了股票监控、新闻爬取、分析师数据采集和IPTV管理等多个功能模块，通过Docker容器化部署，使用Nginx作为统一的Web访问入口。
